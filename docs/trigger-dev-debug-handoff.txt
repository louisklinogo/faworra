# Trigger.dev .env Loading Issue ‚Äî Resolved

## Status
Fixed. `trigger dev` loads the project ref correctly; use `"dev": "trigger dev"` in `packages/jobs/package.json` (Midday parity).

## Root Cause
The Trigger.dev CLI merges `.env` for its own options but does **not** put those variables into `process.env` before Jiti evaluates `trigger.config.ts`. The line `project: process.env.TRIGGER_PROJECT_ID` therefore saw `undefined`, which surfaced as `Project not found: undefined`. The `--env-file .env` flag did not change that, because it only feeds the CLI merged env, not `process.env` at config evaluation time.

The `dev` command also reads `TRIGGER_PROJECT_REF` for CLI overrides; Faworra `.env` files typically only set `TRIGGER_PROJECT_ID`. That is fine once `process.env` is populated before `defineConfig` runs.

## Fix (in repo)
1. **`packages/jobs/trigger.config.ts`** ‚Äî Call `dotenv.config()` with `path` next to the config file (`dirname(fileURLToPath(import.meta.url))`) and `{ quiet: true }`, **before** `defineConfig(...)`.
2. **`packages/jobs/package.json`** ‚Äî Add direct dependency `"dotenv": "catalog:"` (and `bun install`).
